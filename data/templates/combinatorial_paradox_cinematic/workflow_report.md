## 最值得复用的模板规律

| 类别 | 结果 |
|---|---|
| 结构规律 | 不是“先概念定义”，而是“先做不可能感，再做破解路径” |
| 页面规律 | 每个页面都只服务一个理解动作：设题、给规则、演示机制、压出概率、抽象方法 |
| 节奏规律 | hook 高压，body 讲机制时放慢但不静止，结论处再收紧 |
| 视觉规律 | 物件阵列与关系线是主角，人物和现实镜头只是调味和落点 |
| 音字规律 | 字幕按句组换，转场点整体换句；音频最终必须反向驱动字幕时间轴 |

| 问题 | 回答 |

# 单独说明

| 问题 | 结果 |
|---|---|
| 1. 这次是否复用了任何旧模板骨架 | 没有复用旧模板骨架。 |
| 2. 如果没有复用，新建了哪些 scene 类型 | `ParadoxColdOpenScene`、`RuleTheaterScene`、`MechanismGridScene`、`PairCascadeScene`、`ProbabilityBoardScene`、`ReflectiveCloseScene`。 |
| 3. 如果有部分复用，复用了哪些基础能力 | 仅复用 `SubtitleTrack`、`AudioTrackLayer`、字体系统、基础动效函数、图片加载能力。 |
| 4. 哪些页面系统已经被替换 | 旧的 comic 页、trading HUD 页、psych 关键词页、dossier 页、archive 页、documentary scene 骨架全部不使用。 |
| 5. 为什么最终模板不会看起来像现有模板 | 因为它的主语不是卡片、面板、档案或 HUD，而是“对象阵列与关系结构的电影化演出”。页面母版、镜头语法、scene 职责都重新定义了。 |
| 6. 这个模板未来最适合批量复用的方向是什么 | 所有需要“反直觉开场 + 结构机制揭示 + 可计算结果落点”的数学/概率/系统思维短中视频。 |
